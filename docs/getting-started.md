# Sultan Social API | Quick Start & Integration Guide

Panduan ini menjelaskan alur kerja lengkap (End-to-End) untuk menggunakan **General Social Publisher API Core** bagi pengembang atau pemilik Dashboard.

## 1. Persiapan Infrastruktur
Pastikan server Anda sudah menginstal:
- **Bun Runtime** (Wajib)
- **Node.js v20+** (As fallback)

## 2. Alur Penggunaan API (Workflow)

### Langkah 1: Registrasi Platform
Dapatkan API Key dan Access Token dari masing-masing portal developer:
- [Meta for Developers](https://developers.facebook.com/) (FB, IG, Threads)
- [X Developer Portal](https://developer.twitter.com/en/portal/dashboard) (Twitter)

### Langkah 2: Verifikasi Koneksi
Sebelum melakukan posting, selalu gunakan endpoint `/verify` untuk memastikan kredensial valid dan tidak expired.
- **FB:** `POST /api/v1/facebook/verify`
- **IG:** `POST /api/v1/instagram/verify`
- **X:** `POST /api/v1/x/verify`
- **Threads:** `POST /api/v1/threads/verify`

### Langkah 3: Eksekusi Posting
Kirim konten Anda (Teks/Media) ke endpoint `/post` masing-masing platform.
> **Tips:** Gunakan array `media_urls` untuk mengirim gambar. Server akan otomatis mengunduh dan memproses gambar tersebut.

---

## 3. Sistem Keamanan & Komersialisasi (SaaS Ready)
Sistem ini dirancang untuk mendukung fitur **Subscription** dengan menyertakan header berikut pada setiap request:

| Header | Deskripsi | Kegunaan |
| :--- | :--- | :--- |
| `x-license-key` | Kode Lisensi unik per user. | Validasi status langganan & kuota. |
| `x-device-id` | Hash unik Hardware user (HWID). | Mekanisme kunci akun ke satu perangkat fisik. |
