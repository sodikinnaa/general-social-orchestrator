# X (Twitter) API Setup Tutorial (Step-by-Step)

Panduan ini ditujukan bagi user atau pengembang yang ingin mendapatkan 4 kunci sakti (API Key, API Secret, Access Token, Access Secret) untuk menggunakan **General Social Publisher API**.

## Langkah 1: Registrasi Portal Developer
1. Buka [X Developer Portal](https://developer.twitter.com/en/portal/dashboard).
2. Login dengan akun X Anda.
3. Klik tombol **"Sign up for Free Account"** (Jika belum memiliki akun Developer).

## Langkah 2: Membuat Project & App
1. Di Dashboard, klik **"Create Project"**.
2. Masukkan nama Project (Misal: *Sultan Social Hub*).
3. Buat **"App"** baru di dalam Project tersebut.
4. Pilih **"Production"** sebagai App Environment.

## Langkah 3: Konfigurasi Izin (Wajib!)
Secara default, aplikasi X hanya memiliki izin **Read**. Mas **WAJIB** mengubahnya ke **Read and Write** agar bisa posting:
1. Klik tab **"Settings"** pada aplikasi Anda.
2. Cari section **"User authentication settings"** dan klik **"Set up"**.
3. **App permissions:** Pilih **Read and Write**.
4. **Type of App:** Pilih **Web App, Native App, or Crawler**.
5. **App info:** 
   - Callback URL: `http://localhost` (Bisa dummy).
   - Website URL: `https://example.com` (Bisa dummy).
6. Klik **"Save"**.

## Langkah 4: Mengambil Kunci API (API Keys)
1. Klik tab **"Keys and Tokens"** di Dashboard aplikasi Mas.
2. Di bagian **"Consumer Keys"**, klik **"Generate"** (Atau *Regenerate* jika sudah ada).
3. Mas akan mendapatkan:
   - **`API Key`**
   - **`API Key Secret`**
   - **SIMPAN BAIK-BAIK!** (Mas tidak akan bisa melihat kodenya lagi).

## Langkah 5: Mengambil Token Akses (Access Tokens)
1. Masih di tab **"Keys and Tokens"**.
2. Di bagian **"Authentication Tokens"** -> **"Access Token and Secret"**, klik **"Generate"**.
3. Mas akan mendapatkan:
   - **`Access Token`**
   - **`Access Token Secret`**
4. **PENTING:** Jika Mas baru saja mengubah izin ke *Read and Write* di Langkah 3, Mas **WAJIB** menekan tombol *Regenerate* agar token yang baru memiliki izin posting.

---

### 📡 Cara Mengetes di Sultan Social API:
Gunakan request **`Verify Credentials`** di Postman:
- **URL:** `POST {{baseUrl}}/api/v1/x/verify`
- **Body:** Masukkan 4 kunci di atas ke dalam JSON.
- **Hasil:** Jika sukses, API akan membalas dengan handle `@username` Mas.
