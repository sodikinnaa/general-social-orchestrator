# Facebook Posting API Documentation

Dokumentasi ini merinci penggunaan endpoint Facebook pada **General Social Publisher API**.

## Endpoint: Post to Feed
Digunakan untuk mengirim status teks, foto tunggal, atau album foto (multi-photo) ke Fanspage Facebook.

- **Method:** `POST`
- **URL:** `{{baseUrl}}/api/v1/facebook/post`
- **Headers:**
  - `Content-Type: application/json`
  - `x-license-key`: (Optional) Sultan License Key.
  - `x-device-id`: (Optional) Hardware ID.

### 1. Payload: Text Only
Kirim status berupa teks saja.
```json
{
    "page_id": "1234567890",
    "access_token": "YOUR_PAGE_ACCESS_TOKEN",
    "message": "Hello World! Ini adalah postingan teks dari API."
}
```

### 2. Payload: Single Photo
Kirim satu foto dengan caption.
```json
{
    "page_id": "1234567890",
    "access_token": "YOUR_PAGE_ACCESS_TOKEN",
    "message": "Caption foto keren!",
    "media_urls": [
        "https://example.com/image.jpg"
    ]
}
```

### 3. Payload: Multi-Photo (Album)
Kirim beberapa foto sekaligus dalam satu postingan. API akan otomatis mengunggahnya sebagai album.
```json
{
    "page_id": "1234567890",
    "access_token": "YOUR_PAGE_ACCESS_TOKEN",
    "message": "Kumpulan foto dokumentasi acara.",
    "media_urls": [
        "https://example.com/photo1.jpg",
        "https://example.com/photo2.jpg",
        "https://example.com/photo3.jpg"
    ]
}
```

## Endpoint: List Managed Pages
Digunakan untuk mendapatkan daftar Fanspage yang dikelola oleh user, lengkap dengan **Page ID** dan **Page Access Token**.

- **Method:** `POST`
- **URL:** `{{baseUrl}}/api/v1/facebook/pages`
- **Payload:**
```json
{
    "access_token": "YOUR_USER_ACCESS_TOKEN"
}
```

## 🛠️ Izin & Fitur Utama (Permissions)

Agar aplikasi dapat berinteraksi dengan Facebook API, pastikan aplikasi Anda di **Meta for Developers** memiliki izin berikut:

1.  **`pages_manage_posts`** (Wajib): Memberikan akses untuk membuat postingan ke Halaman.
2.  **`pages_read_engagement`**: Diperlukan untuk membaca interaksi dasar.
3.  **`pages_show_list`**: Diperlukan agar API dapat melihat daftar Halaman yang dikelola.
4.  **`public_profile`**: Diperlukan untuk identitas dasar user.

Langkah aktivasi:
- Buka Meta Developer Portal -> Use Cases -> Customize -> **Kelola Halaman** -> Izin dan fitur.

## Tips & Troubleshooting

### 🔑 Cara Mendapatkan Token yang Benar
Sistem ini membutuhkan **Page Access Token**, bukan *User Access Token*.
1. Gunakan endpoint `/api/v1/facebook/pages` dengan User Token Anda.
2. Cari nama Fanspage Anda di hasil respon.
3. Gunakan field `access_token` yang ada di dalam objek Fanspage tersebut untuk melakukan posting.

### ⚠️ Error Common: (#200) Permission Denied
Jika muncul error ini, pastikan:
1. Anda menargetkan **Page ID**, bukan Profil Pribadi.
2. Aplikasi Meta Anda sudah memiliki izin `pages_manage_posts`.
3. Token yang digunakan adalah **Page Access Token**.
